# 2026-05-30 — 저녁 식당 사전 넷예약(大鵬·まんざら亭) 채택 + 川床 메모 정정

## Context (왜)

- 사용자가 "현장(워크인) 대기는 리스크 크다"고 명시 — 4인 가족(시부모 2인 고령자 포함)이라 줄서기·대기 회피가 우선.
- 확정 일정의 저녁 2곳이 인기점이라 워크인 대기 가능성이 높음:
  - 5/31(일) **中華 大鵬** (타베로그 3.76·미쉐린 빕구르망) — 기사 제목에 "予約至難·行列必至" 등장.
  - 6/1(월) **まんざら亭 先斗町店** (창작 교토요리) — 폰토초 인기 居酒屋.
- 실시간 리서치(2026-05-30, WebSearch/WebFetch)로 두 곳의 예약 채널을 확인:
  - 大鵬: **AutoReserve 넷예약** 가능(슬롯 17:30·19:00, 전일 18:00 마감) / 전화 075-822-5598. 일요일 디너 영업, 화·격주 수 휴무, 30석·개실 없음.
  - まんざら亭: **楽天ぐるなび·ヒトサラ 넷예약** 가능 / 전화 075-212-0028. 매일 영업, 4인 일반석 가능.
- 부수 확인: itinerary 메모의 "まんざら亭 川床 6~9월 가능"이 부정확. 공식 가모가와 납량상(納涼床) 점포목록(kyoto-yuka.com 先斗町 구역)에 **先斗町店 미등재** — 強床席은 別店(木屋町 계열)일 가능성, 先斗町店 자체는 미운영으로 판단.
- 도착 동선 제약: 5/31 KIX 15:15 도착 → 시오 체크인 17:45. 大鵬 예약 슬롯 17:30은 체크인 직후라 **불가** → 19:00이 유일 현실 슬롯(원래 계획 18:30과 근접).
- 그 외 식사(eX cafe 6/1 점심·中村商店 라멘 6/2 점심·MACCHA HOUSE 6/2)는 예약 개념 없음(워크인·식권·카운터)으로 확인 — 별도 예약 불필요.

## Decision (무엇)

- 大鵬(5/31)·まんざら亭(6/1) 두 저녁을 **사전 넷예약**으로 확보하기로 하고, `booking-checklist.json`에 `dining_reservations` 항목(상태 `예약중`)으로 추적한다.
- 大鵬 일정 시각을 **18:30 → 19:00**으로 조정(체크인 17:45 직후 17:30 슬롯 불가 반영).
- まんざら亭 food_quality 메모의 **川床 문구를 정정**(先斗町店 납량상 미운영 명시).
- 채택하지 않은 대안:
  - 워크인 — 사용자가 명시적으로 배제(대기 리스크).
  - 전화 예약 — 일본어·국제전화 부담으로 차선. 넷예약 가능하므로 보조 수단으로만 표기.
  - 静家(유바)를 1순위로 — 9석뿐이라 4인 수용·예약 난이도가 더 높음. 大鵬 만석 시 백업으로만 유지.

## Consequences (그래서)

- 긍정: 두 저녁 대기 리스크 제거, 전화 없이 넷예약으로 확정 가능. 일정 시각이 실제 예약 슬롯과 정합.
- 부정·트레이드오프: 실제 예약 클릭(소연/영욱 계정)은 사람이 수행해야 하며, 예약번호 확보 전까지는 `예약중` 상태. 大鵬 19:00은 전일 18:00 마감이라 5/30 중 처리 권장.
- 후속 행동: 두 곳 넷예약 완료 시 `dining_reservations`를 `확정`으로 승격 + `reference`(예약번호) 기입 + 신규 일지. 静家를 백업으로 실제 잡을 경우 별도 기록.
- 영향 받은 파일·데이터:
  - `data/booking-checklist.json` — `dining_reservations` 신규 항목.
  - `data/itinerary.json` — 大鵬 time 19:00·예약 채널 메모, まんざら亭 川床 정정·예약 채널 메모.
