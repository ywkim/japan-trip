# 2026-05-31 — 5/31 첫날 저녁을 쿄 우나와 본점(장어) 단출 왕복으로 변경 (PR #73 충돌 해소)

## Status

Accepted — PR #73("첫날 야사카 신사 야경 + 가모강 산책")의 5/31 저녁안을 Supersede. PR #73 머지 충돌(`data/itinerary.json`·`docs/kyoto-itinerary-may31-jun3-2026.md`·`tests/test_build_index.py`) 해소 커밋.

## Context (왜)

- PR #73은 5/31 저녁을 "기온 원정 — 牛カツ勝牛(규카츠) → 야사카 신사 야경 → 가모강 산책"으로 추가했다. 그 사이 `main`은 독립적으로 같은 슬롯을 "폰토초 enen 야끼니꾸 + 가모강 야경 산책"으로 바꿔 머지했다(#73 base 이후). 두 안이 같은 저녁 슬롯을 두고 **의미 충돌**(상호 배타적 식당·동선).
- 사용자(소연)가 충돌 해소 과정에서 5/31 저녁 방침을 다시 정했다: ① 동산 원정(야사카)·가모강 산책 **생략**, ② 저녁은 **시오 근처에서 시버스로 시내(四条)까지 가 쿄 우나와 본점(京うな和本店) 장어만 먹고 바로 복귀**하는 단출한 동선. 오후 도착(KIX 15:15)·시부모(고령 2인) 동반이라 도착 당일 동선 부담을 최소화하려는 의도.
- 실시간 리서치(2026-05-31, 환경상 maps/tabelog 직접 열람 차단 → 검색·공식·구르나비·핫페퍼 교차): 京うな和本店 四条堺町 = 나고야 계열 장어요리 전문점(2020 교토 개업), 우나기주·히츠마부시, 영업 11:00~20:30(L.O.20:00), 디너 1인 약 ¥5,500, 시조카라스마(四条烏丸)·시조카와라마치(四条河原町)역 도보 5분. 같은 이름의 神宮道점이 별도 존재.

## Decision (무엇)

- 5/31 저녁 = **京うな和本店 四条堺町 장어 1곳**, 시오↔시내 **시버스(균일 ¥230) 왕복**으로 단순화. 야사카 신사·가모강 산책 항목은 확정 코스에서 제거(동산 지구는 6/2 오후 유지).
- 충돌 해소 방식:
  - `data/itinerary.json`·`docs/...md`: PR #73이 두 파일에서 5/31 day만 건드렸으므로 **main 버전을 취한 뒤** 저녁(enen·가모강 2항목)을 **쿄 우나와 1항목**으로 재작성. 장소명 한자 병기는 `{{kamogawa}}` 참조 등 레지스트리 규약 준수(검사 K).
  - `tests/test_build_index.py`: `test_verified_source_shows_tick`는 main의 진화된 버전(source_chip·`source-tick`) 채택. `test_long_food_note_folded_but_rating_kept`는 제거되는 enen 대신 **kadensho("130년 된 교마치야")** 앵커를 유지(=PR #73 측) — 새 동선에 enen이 없어 main 앵커는 깨지므로.
- `arrive_from`은 노선번호 미검증이라 `researched_market_rate`(시버스 균일 ¥230은 교토시 교통국 공식) — 탑승 전 구글맵/시버스 앱 재확인.
- 채택하지 않은 대안: 스키야끼(牛大)안(직전 후보, 사용자가 우나와로 재변경), enen·야사카 유지(사용자가 명시적으로 생략 지시).

## Consequences (그래서)

- 긍정: PR #73이 현재 main과 충돌 없이 머지 가능. 도착 당일 저녁이 시버스 왕복 1회·식사 1곳으로 단순해져 고령자 동선 부담 최소. 빌드·validate(검사 G/I/K/L 포함)·209 테스트 통과.
- 부정·트레이드오프: PR #73의 본래 콘텐츠(야사카 야경·가모강 산책)는 폐기. enen 야끼니꾸안도 폐기. 쿄 우나와 평점(타베로그 점수)은 환경 제약으로 미확보 → `food_quality.rating`에 점수 대신 영업·가격 사실만 출처와 함께 기재.
- 후속: 쿄 우나와 본점 4인 테이블석 예약(영어 가능)·정확한 시버스 노선·요금 현지/구글맵 재확인. 야사카 신사 야경을 원하면 6/2 동산 코스에 편입 검토.
- 영향 받은 파일: `data/itinerary.json`(5/31 day), `docs/kyoto-itinerary-may31-jun3-2026.md`(§1.1·운영 메모), `tests/test_build_index.py`(회귀 가드 2건).
