# 2026-05-26 — 조식 페이지 가독성·가격/메뉴 노출 개선

## Status

Accepted (`2026-05-26-04-breakfast-fold-mobile-readability.md` 후속 — 실제 배포 화면 피드백 반영)

## Context (왜)

- 사용자 피드백(2026-05-26, 배포 스크린샷): "가격이나 메뉴도 안 보이고 가독성이 낮아."
- 두 가지 구체 문제:
  1. **우측정렬 k/v 행의 줄바꿈 파괴**: 아침 3회 표·아침별 권장의 긴 문장이 `.row .v`(`text-align:right`)에 들어가 모바일에서 우측정렬로 4~6줄씩 깨져 읽기 어려움.
  2. **가격·메뉴 비노출**: 메뉴·가격(¥750 등)이 가게 `note` 산문에 묻혀 있고, 직전 PR에서 가게 그룹을 전부 접어 한눈에 안 보임.

## Decision (무엇)

긴 텍스트는 좌측정렬 블록으로, 가격·메뉴는 전용 라인으로, 숙소별 첫 그룹은 펼친 채로 렌더한다.

- `scripts/build_index.py`:
  - 아침 3회·아침별 권장을 `.row`(우측정렬 k/v) → `.bf-item`/`.bf-label`/`.bf-body`(좌측정렬 스택 블록)로 변경.
  - 가게 `menu`를 `.bf-menu` 전용 라인(🍽)으로 노출. `_breakfast_store`가 rows 다음에 menu→note 순서로 렌더.
  - `_breakfast_group(..., open=)` 추가 — 숙소별 **첫(가장 편리) 그룹은 기본 펼침**, 나머지는 접힘.
  - CSS에 `.bf-item`/`.bf-label`/`.bf-body`/`.bf-menu`/`.bf-store` 추가(기존 토큰만 참조 — 새 hex 없음, 검사 H 무영향).
- `data/breakfast.json`: 가게 `note`에 섞여 있던 메뉴·가격을 `menu` 필드로 분리(신규 데이터 아님, §5 리서치 출처 그대로). 동네 끼니·체인은 `menu` 생략.
- 미채택: (a) 전체 그룹 기본 펼침 — 페이지가 다시 길어짐. (b) 가격을 k/v 행으로 — 긴 메뉴명이 우측정렬로 깨짐. (c) 페이지 재정렬 — 이번 범위는 가독성/노출이라 순서 유지.

## Consequences (그래서)

- 긍정: 긴 문장이 좌측정렬로 자연스럽게 읽히고, 가격·메뉴가 🍽 라인으로 즉시 보인다. 첫 그룹 펼침으로 핵심 카페(코메다·小川珈琲 등)를 탭 없이 확인.
- 부정·트레이드오프: 첫 그룹 펼침으로 페이지가 직전보다 약간 길어짐(가독성과의 맞바꿈). `note`↔`menu` 분리로 데이터 항목 증가.
- 후속: 현지에서 메뉴·가격 변동 확인되면 `menu` 갱신(§4 주의 — 출발 전 재확인). 필요 시 메뉴 출처를 가게별로 더 정밀히 연결.
- 영향 파일: `scripts/build_index.py`, `data/breakfast.json`, `tests/test_build_index.py`, 빌드 산출물(공통 CSS로 7 HTML), `CLAUDE.md`.
