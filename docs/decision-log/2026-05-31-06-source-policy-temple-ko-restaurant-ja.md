# 2026-05-31 — 출처 종류 정책: 가게는 일본어 1차, 사찰·관광지는 한국어 블로그

## Status

Accepted

## Context (왜)

6/1 일정 보강(사이호지 문서 링크 연결·료안지 후기 추가)을 논의하던 중, 사용자가 출처 선택의 **명시적 정책**을 요구했다: "사찰은 한국어 출처만, 가게는 일본어 출처만 정책 준수 문서화."

배경이 된 사실들:
- **PR #89 날조 사건**(`docs/decision-log/2026-05-31-04-excafe-manzaratei-fabrication-postmortem.md`): 카페·식당(eX cafe·만자라테이) 정보를 한국어 블로그 기억·추측으로 작성해 가격(당고 세트 ¥1,320)·휴무("월요일 휴무")가 **사실과 달랐다**. Tabelog·ぐるなび 일본어 1차 출처로 교차검증하니 실제는 ¥1,850·`不定休`였다.
- 반면 사찰·관광지(후시미이나리·토후쿠지·금각사·텐류지·사이호지 등)의 `blog_reviews`는 **한국어 네이버 블로그**로 채워져 있고, 이는 "한국 여행자 관점의 실용 팁"(이른 아침 인파 회피·시부모 동반 난이도·6월 분위기·포토스팟)을 전달해 가족 여행에 유용했다.
- 즉 **항목 종류에 따라 적절한 출처가 다르다**. 식당의 "사실"(가격·영업·휴무)과 관광지의 "경험 후기"는 검증 요구 수준이 다르다.

## Decision (무엇)

항목 종류별로 허용 출처를 **차등 고정**하고 CLAUDE.md "실시간 가격 리서치 우선" 절에 명문화한다.

- **식당·카페 → 일본어 1차 출처만**(Tabelog·ぐるなび·공식). `food_quality`·메뉴·가격·영업시간·휴무·주소에 적용. 한국어 블로그의 사실 인용 금지(옛 가격·오역·추측 위험). 검사 I가 food_quality 출처를 강제.
- **사찰·신사·관광지 → 한국어 블로그(네이버 등) 허용**. `blog_reviews` 후기 카드에 적용. blog_reviews는 사실 주장이 아닌 "후기 읽기" 경험 공유 링크이므로 한국어 출처가 오히려 적합. 단 **실제 존재하는 포스트 URL + 실제 이미지 URL만**(날조 금지, 없으면 보류).

이번 PR의 구체 적용:
1. **사이호지(10:30)**: 이미 존재하던 `viz/saihoji.html`(예약·사경·참배료·접근성)에 `link` 필드로 연결. note 끝의 평문 경로(`docs/...md`) 제거.
2. **료안지(15:00) 후기**: 정책상 한국어 네이버 블로그가 정석이나, 현 세션 도구(US 한정 WebSearch)로는 **검증된 네이버 포스트+이미지 URL을 확보하지 못함**(Naver 색인 부족). 날조 대신 **추가 보류** — 사용자 제공 또는 검증 경로 확보 후 진행.

채택하지 않은 대안:
- **전 항목 일본어 1차 출처 통일**: 사찰 실용 팁(한국 여행자 관점)을 잃음. 미채택.
- **전 항목 한국어 블로그 허용**: 식당 가격·휴무 오정보 재발 위험(PR #89 재현). 미채택.
- **료안지 후기를 그럴듯한 네이버 URL로 채움**: 날조 — 본 레포 규칙 정면 위반. 미채택.

## Consequences (그래서)

**긍정**:
- 출처 선택 기준이 명문화되어 후속 세션이 "식당엔 Tabelog, 사찰엔 네이버"를 일관 적용.
- 식당 사실 오정보(PR #89류) 재발 방지 + 사찰 실용 후기의 가치 보존.
- 사이호지 핵심 일정에 참배·예약 문서가 탭으로 연결됨.

**부정·트레이드오프**:
- 료안지 후기는 이번에 미완(보류). 도구 한계로 검증된 한국어 출처 확보가 막힘 — 사용자 입력 의존.

**후속 행동**:
- 료안지 `blog_reviews`: 사용자가 네이버 블로그 링크 제공 시 추가 → `fetch_assets.py`로 이미지 자가호스팅.
- 신규 항목 추가 시 본 정책의 종류별 출처 표를 따른다.

**영향 받은 파일**:
- `data/itinerary.json` (사이호지 `link` 추가, note 정리)
- `CLAUDE.md` ("출처 종류 정책" 절 신설)
- 본 일지
