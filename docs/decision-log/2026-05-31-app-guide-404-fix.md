# 2026-05-31 — 앱 가이드 링크 404 수정 (doc_link_html / checklist_card in_viz 버그)

## Status

Accepted

## Context (왜)

- `index.html`(루트)에서 "앱 가이드 ↗" 버튼을 탭하면 Vercel에서 404 반환
- `checklist_card` 함수가 `in_viz=True`를 하드코딩하여, 루트·viz 두 컨텍스트 모두에서 viz 상대경로(`essential-iphone-apps.html`)를 생성
- `index.html`은 루트(`/`)이므로 실제 파일 경로는 `viz/essential-iphone-apps.html` → `/essential-iphone-apps.html`은 존재하지 않아 404
- 같은 이유로 `doc_link_html`도 `itinerary.json`의 `breakfast.html` 등 viz-상대 경로를 루트에서 그대로 출력 → 동일 버그 잠재

## Decision (무엇)

- `checklist_card(it, in_viz: bool = False)` 파라미터 추가 → `doc_href` 호출 시 전달
  - `build_checklist` (viz/checklist.html)에서: `in_viz=True` 명시
  - `card_checklist` (index.html)에서: 기본값 `False` 유지
- `doc_link_html(link, in_viz: bool = False)` 파라미터 추가 → `in_viz=False`이고 URL이 viz-상대 `.html`이면 `viz/` 접두어 보정
  - `build_itinerary`, `build_itinerary_table` (viz 페이지)에서: `in_viz=True` 명시
  - `card_itinerary` (index.html)에서: 기본값 `False` 유지
- 미채택 대안: 데이터(`itinerary.json`)의 URL을 `viz/breakfast.html`로 변경 → 정본 스키마 변경이 필요해 파급 범위가 크고, 렌더 레이어에서 처리하는 것이 더 일관적

## Consequences (그래서)

- index.html의 "앱 가이드 ↗" 링크가 `viz/essential-iphone-apps.html`로 정상 렌더
- `breakfast.html`, `research.html`, `transit-pass.html` 등 checklist·itinerary의 모든 doc-link가 루트에서도 올바른 경로 생성
- viz 페이지(viz/checklist.html, viz/itinerary.html, viz/itinerary-table.html) 동작 변경 없음
- TDD: 3개 신규 테스트 (`ChecklistCardDocLinkTests`) 추가 + 192개 전체 통과
- 영향 파일:
  - `scripts/build_index.py` (doc_link_html·checklist_card 시그니처 + 5개 호출 지점)
  - `tests/test_build_index.py` (3개 신규 테스트)
  - `docs/decision-log/2026-05-31-app-guide-404-fix.md` (이 파일)
