# 2026-06-02 — PR #73 충돌 해소: 5/31 저녁 교고쿠 카네요(main) 유지, 쿄 우나와·야사카 안 폐기

## Status

## Context (왜)

- PR #73(`claude/yasaka-shrine-itinerary-IbTT5`)은 5/31 저녁 슬롯을 여러 차례 재설계했다: 야사카 신사 야경 + 가모강 산책 → (충돌 해소 과정에서) **쿄 우나와 본점(京うな和本店) 장어 단출 왕복**.
- PR #73이 마지막으로 main을 병합한 시점(2026-05-31 08:14 UTC)에 main의 5/31 저녁은 "폰토초 enen 야끼니꾸"였고, PR #73은 이를 superseding하는 것으로 설계됐다.
- 그러나 그 **약 5시간 뒤**(2026-05-31 13:00 UTC) PR #88이 main에 머지되며 같은 5/31 저녁 슬롯을 **교고쿠 카네요(京極かねよ) 우나기 킨시동 + 신쿄고쿠 상점가 야경 산책**으로 교체했다. 카네요 안은 예약 준비·전체 메뉴·일본어 후기 번역 페이지 2개(`viz/kaneyo-review.html`·`viz/shinkyogoku-review.html`)·places 레지스트리 참조·교통 타임라인까지 갖춘 더 진화한 안이다.
- 결과적으로 PR #73(쿄 우나와)과 현재 main(카네요)이 같은 저녁 슬롯을 두고 **상호 배타적으로 충돌**(`data/itinerary.json` 5/31 day·`docs/kyoto-itinerary-may31-jun3-2026.md` §1.1·§3·`tests/test_build_index.py` food-note 폴딩 회귀 가드).
- 사용자(소연)가 충돌 해소 방향을 직접 결정: **PR #73의 변경을 생략하고 카네요(main)를 유지**한다. (둘 다 우나기 저녁이며, 카네요가 더 나중에 내린 결정이자 예약·후기까지 갖춘 안.)

## Decision (무엇)

- 충돌 3개 파일을 모두 **main(카네요) 측으로 해소**한다(`git checkout --ours`). 즉 5/31 저녁은 교고쿠 카네요 우나기 킨시동 + 신쿄고쿠 산책을 그대로 유지하고, PR #73이 도입하려던 쿄 우나와·야사카·가모강 동선은 채택하지 않는다.
- PR #73이 추가했던 폐기 라인 일지 3건(`2026-05-30-day1-yasaka-night-kamo.md`·`2026-05-30-04-merge-main-yasaka-night.md`·`2026-05-31-05-day1-dinner-kyo-unawa.md`)은 **본 머지 결과에서 제거**한다. 이들은 채택하지 않은 안을 "Accepted"로 단언해 레포 실상(카네요)과 모순되기 때문이며, 탐색 경위는 본 일지가 대신 보존한다.
- 채택하지 않은 대안: ① 쿄 우나와로 교체(=PR #73 원안 — 더 나중의 카네요 결정·예약·후기 페이지를 되돌리게 되어 기각), ② PR #73 닫기(사용자가 유지 선택).
