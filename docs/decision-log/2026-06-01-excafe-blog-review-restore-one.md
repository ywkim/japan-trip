# 2026-06-01 — eX cafe blog_reviews 0번 카드 복원 (1장) + 분석 정정

## Status

Accepted. `2026-05-31-10-excafe-blog-reviews-not-restored-env-block.md`(미복원 확정)를 **부분 정정** — 검증 가능한 1장은 복원.

## Context (왜)

사용자 질문: "왜 origin/main의 정상 블로그를 삭제했나." 이를 계기로 origin/main eX cafe 15장을 **카드 단위로 재분해**했다(2026-05-31~06-01).

발견:
- **URL 종류**: 실제 글(`/<id>/<글번호>`) **1**(0번) + 블로거 홈 5(1~5번) + 검색 페이지 9(6~14번).
- **댓글**: 15장 전부 `[가게명] 특징 요약` **합성문**(실제 글 발췌 0).
- **URL·이미지 작성자 대조**: 0번은 URL=`ramu0527`인데 이미지(`IMG_9263.JPG`)는 `m0846`의 2019년 사진 — **출처 불일치**(조립 카드). 역설적으로 1~5번은 URL·이미지 작성자가 같으나(홈 URL) 특정 글로는 연결 안 됨.

즉 "URL·이미지·댓글이 모두 한 실제 글에서 온" 카드는 **0개**였다. 앞 턴들에서 "1장은 실제 포스트"라 뭉뚱그린 표현은 부정확했다(0번은 URL만 실제, 이미지는 타인 글).

내 08 제거의 실제 문제: 카드를 **옥석 구분 없이 전량 삭제**했고, "정책 위반"이라는 (09에서 사용자가 정정한) 잘못된 근거로 정당화했다. 0번은 최소한 **검증 가능한 실제 글 URL**을 가졌는데 함께 지웠다.

환경 제약은 그대로다(`*.naver.com` egress 차단 — `2026-05-31-10` 참조). 새 후기 글을 찾아 검증할 수는 없으나, **origin/main에 이미 있던 0번은 URL이 특정 글이고 이미지가 자가호스팅된 실제 카페 사진**이라 추가 네트워크 없이 복원 가능.

## Decision (무엇)

**eX cafe blog_reviews를 0번 1장으로 복원**한다(origin/main 원본 그대로: url `ramu0527/223959935624` + 이미지 + 댓글). 사용자가 출처 불일치(이미지=타인 글)를 인지하고도 "0번 그대로 복원"을 선택.

- 이미지 매핑 복구(`fetch_assets.py`, local-image-map 36→37, `--check` 0 missing).
- 게이트 통과: 0번 URL은 특정 포스트라 `test_naver_blog_reviews_link_to_specific_posts` 통과, 이미지 자가호스팅으로 `test_every_blog_image_is_self_hosted` 통과.
- 나머지 14장(검색 9·홈 5)은 **미복원** — 게이트가 차단하는 날조 시그니처.

채택하지 않은 대안:
- **15장 전량 복원**: 검색·홈 URL은 게이트 위반(특정 글 아님). 미채택.
- **0번도 미복원(현 상태)**: 출처 불일치 우려는 타당하나, 사용자가 검증 가능한 실제 글 URL 1장 유지를 선택. 미채택.
- **URL만 복원(이미지 제거)**: 이미지가 실제 카페 사진이고 자가호스팅됨 — 굳이 비울 이유 없음. 미채택.
