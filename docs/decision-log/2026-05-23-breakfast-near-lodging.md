# 2026-05-23 — 숙소 인근 조식 옵션 리서치

## Status

Accepted

## Context (왜)

- 사용자 지시(2026-05-23): "여행 중 조식이 중요하다 — 숙소에서의 거리와 영업시간을 조사하라."
- 4인 중 시부모 2인 동반 → 아침을 거르면 당일 컨디션·페이스에 직접 영향. 조식 동선이 일정 출발 시각과 충돌할 수 있음.
- 조식이 필요한 아침 3회의 제약이 서로 다름:
  - 6/1(월) 시오: 09:00 출발 — 여유 있음.
  - 6/2(화) 시오: 07:30 후시미이나리 조기 출발 — 07:00 오픈 매장도 빠듯.
  - 6/3(수) 카덴쇼: 새벽 공항 출발(KIX 10:05 발) — 대부분 매장 오픈 전. 카덴쇼는 조식 미포함 예약.
- 가격·운영 정보는 기억 추측 금지(CLAUDE.md "실시간 가격 리서치 우선") → WebSearch/WebFetch로 영업시간·거리 교차 검증.

## Decision (무엇)

- `docs/breakfast-near-lodging.md` 신규 작성 — 조식 정보의 단일 출처(standalone 리서치 문서, transit-pass·airbnb 문서와 동일 패턴).
- 숙소 2곳(시오/니조역, 카덴쇼/교토역) 별로 가게명·거리·영업시간·휴무·출처 정리 + 아침 3회별 권장.
- 대안 비교: 일정(`data/itinerary.json`)에 조식 항목을 직접 추가하는 안은 보류 — 영업시간은 변동성이 큰 시세성 정보라 빌드 산출물(viz/index)에 박제하면 묵은 데이터 위험. 일정과 분리된 참조 문서로 두고 출발 전 재확인하는 운영 방식을 채택.

## Consequences (그래서)

- 긍정: 아침별 출발 시각에 맞춘 구체적 선택지 확보. 핵심안 — 6/1·6/2는 코메다 京都二条駅前店(니조역 2분, 07:00 오픈·무료 모닝서비스), 6/3은 전날 편의점 조달 + 하루카 차내·KIX 취식.
- 부정·트레이드오프: 영업시간은 리서치 시점(2026-05-23) 스냅샷 — 출발 전 Google Maps 재확인 필요(문서 §4에 명시). 마스산 목요일 휴무 등 변동 요소 존재.
- 후속: 발권 후 6/3 하루카 실제 출발 시각이 확정되면(07:00 이후 가능 여부) 교토역 매장 모닝 가능 여부 재판단.
- 영향 받은 파일: `docs/breakfast-near-lodging.md`(신규), `README.md`·`CLAUDE.md`(문서 색인·디렉토리 트리 갱신).
