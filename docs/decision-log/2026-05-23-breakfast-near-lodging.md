# 2026-05-23 — 숙소 인근 조식 옵션 리서치

## Status

Accepted

## Context (왜)

- 사용자 지시(2026-05-23): "여행 중 조식이 중요하다 — 숙소에서의 거리와 영업시간을 조사하라."
- 4인 중 시부모 2인 동반 → 아침을 거르면 당일 컨디션·페이스에 직접 영향. 조식 동선이 일정 출발 시각과 충돌할 수 있음.
- 조식이 필요한 아침 3회의 제약이 서로 다름:
  - 6/1(월) 시오: 09:00 출발 — 여유 있음.
  - 6/2(화) 시오: 07:30 후시미이나리 조기 출발 — 07:00 오픈 매장도 빠듯.
  - 6/3(수) 카덴쇼: 새벽 공항 출발(KIX 10:05 발) — 대부분 매장 오픈 전. 카덴쇼는 조식 미포함 예약.
- 가격·운영 정보는 기억 추측 금지(CLAUDE.md "실시간 가격 리서치 우선") → WebSearch/WebFetch로 영업시간·거리 교차 검증.

## Decision (무엇)

- 2층 구조 채택:
  1. **`docs/breakfast-near-lodging.md`** — 조식 옵션의 단일 출처(가게명·니조역/교토역 기준 거리·영업시간·휴무·출처 + 아침 3회별 권장). standalone 리서치 문서(transit-pass·airbnb 문서와 동일 패턴).
  2. **`data/itinerary.json`** 확정 코스(days) 6/1·6/2·6/3 각 날 첫 item으로 **간결한 조식 슬롯** 추가 — viz/index에서 "언제·기본 선택지"를 노출하고 상세는 위 문서로 위임(슬롯 note에서 참조). `pending`에 "조식 식자재 사전 구비" 추가.
- 초기엔 itinerary 임베드를 보류(영업시간 변동성→빌드 산출물 박제 위험)했으나, 병렬 작업 PR #41이 슬롯 임베드의 실용성을 입증 → 두 접근을 통합. **변동성 리스크는 슬롯에 시각·고정 사실(자취·편의점 정본)만 두고 가변 영업시간은 참조 문서로 분리**하여 완화.
- 정정: PR #41의 6/1 "오가와커피 본점"(서쿄고쿠 소재, 니조 무관)을 **코메다 京都二条駅前店(니조역 도보 2분, 07:00, 무료 모닝서비스)**로 교체 — ekiten 검증.
- 채택하지 않은 대안: 카덴쇼 조식 추가 주문(6/3 새벽 출국 동선 우선 기존 결정 유지) / 6/2·6/3 카페 외식(영업시간 07:00~ 미스매치). `route_candidates`는 동선 스케치라 미반영.
