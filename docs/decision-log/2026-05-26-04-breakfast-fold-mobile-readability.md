# 2026-05-26 — 조식 페이지 가게 그룹을 접기(fold)로 모바일 가독성 개선

## Status

Accepted (`2026-05-26-02`·`-03` 후속 — 같은 조식 페이지의 표시 밀도 조정)

## Context (왜)

- 사용자 지시(2026-05-26): "접기(fold)로 모바일 가독성." `viz/breakfast.html`이 시오 3개 그룹·카덴쇼 2개 그룹의 가게(총 14곳)를 모두 펼쳐 렌더 → 모바일에서 세로로 매우 길어 핵심(아침별 권장)이 묻혔다.
- 레포에는 이미 장문 블록을 `fold(summary, detail)`로 접는 모바일 가독성 규칙(CLAUDE.md)이 있다 — 이동 경로·예약 메모·교통패스 추천 등. 조식 가게 목록은 같은 성격의 참조성 장문이다.

## Decision (무엇)

조식 가게 그룹과 주의 블록을 `fold()`로 접고, 오리엔테이션·실행 정보는 펼친 채 둔다.

- `scripts/build_index.py`에 `_breakfast_group(g, map_area)` 추가 — 가게 그룹을 "라벨 · N곳"(또는 "· N항목") 요약 + 접힌 상세(`<details class="leg">`)로 렌더.
- 주의(영업시간 변동) 블록도 "⚠ … 출발 전 Google Maps 재확인" 요약으로 접음. 출처는 기존대로 접힘 유지.
- **펼친 채 유지**: 아침 3회 표(오리엔테이션 요약), 아침별 권장(실행 핵심) — 탭 없이 바로 보이게.
- 미채택: (a) 전체 카드 접기 — 숙소 이름·접근성까지 숨겨 맥락 상실. (b) 권장까지 접기 — 가장 자주 보는 실행 정보라 펼침 유지. (c) 페이지 재정렬(권장을 맨 위로) — 이번 범위는 접기이므로 순서는 유지.

## Consequences (그래서)

- 긍정: 모바일 기본 화면이 짧아지고, 아침별 권장이 가게 목록에 묻히지 않는다. 가게 상세·지도 링크는 한 번 탭으로 접근. 레포의 기존 fold 패턴과 일관.
- 부정·트레이드오프: 가게를 보려면 그룹을 한 번 펼쳐야 함(탭 1회 추가). `<details>` 기본 닫힘이라 데스크톱에선 정보 밀도가 약간 낮아짐.
- 후속: 필요 시 "가장 편리" 그룹을 기본 펼침(`open=True`)으로 둘지 검토. 페이지 순서(권장 상단) 재배치는 별도 판단.
- 영향 파일: `scripts/build_index.py`, `tests/test_build_index.py`, `viz/breakfast.html`(빌드), `CLAUDE.md`.
