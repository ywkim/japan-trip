# 2026-05-31 — eX cafe blog_reviews 날조 발견·제거 (카드는 가짜였다)

## Status

Accepted (supersedes 일부: `2026-05-31-07-blog-image-vanish-postmortem.md`의 "텍스트 카드로 유지" 결정)

## Context (왜)

직전 일지 `2026-05-31-07`에서 eX cafe(6/1 12:00) blog_reviews 카드의 "있다가 없다가" 회귀를 mixed-content/`onerror` 문제로 진단하고, 죽은 이미지는 `img=""`로 비워 **텍스트 카드로 유지**하기로 했다. 배포 후 사용자가 그 텍스트 카드들을 보고 **"이건 다르잖아"** 라고 지적했다 — 진짜 사찰 후기 카드와 **질이 다르다**는 것.

정밀 대조 결과, **eX cafe blog_reviews 15장은 대부분 날조**였다(2026-05-31 검증):

| 항목 | 종류 | 총 | 실제 포스트 | 검색결과 페이지 | 블로그 홈 |
|---|---|---|---|---|---|
| 죽림길 | 관광지 | 2 | **2** | 0 | 0 |
| 텐류지 | 사찰 | 2 | **2** | 0 | 0 |
| 사이호지 | 사찰 | 2 | **2** | 0 | 0 |
| 금각사 | 사찰 | 3 | **3** | 0 | 0 |
| **eX cafe** | **카페** | **15** | **1** | **9** | **5** |

- **댓글이 후기 발췌가 아니다**: `[가게명] 특징1. 특징2 — tabelog 점수/영업시간` 형식의 **합성 설명문**. 한 장(8번)은 댓글이 아예 `"…네이버 블로그 후기 검색"`(지시문/플레이스홀더)이었다.
- **URL이 가짜다**: 9장은 `search.naver.com/search.naver?where=blog&query=…`(**검색 결과 페이지** — "후기 읽기 →"가 실제 후기가 아니라 검색창으로 연결), 5장은 `blog.naver.com/<id>`(특정 포스트가 아닌 **블로그 홈** — 자가호스팅 이미지 파일명 경로에서 추출한 작성자 ID로 만든 가짜 링크). 실제 포스트는 1장뿐.
- **출처 정책 위반**: `2026-05-31-06-source-policy-temple-ko-restaurant-ja.md`는 **식당·카페는 일본어 1차 출처(Tabelog·ぐるなび)만**, 한국어 blog_reviews는 **사찰·관광지 전용**으로 못박았다. eX cafe는 카페이므로 한국어 blog_reviews 자체가 금지 대상이다.

즉 사찰 4항목의 blog_reviews는 **실제 네이버 포스트로 정상**이었고(정책 준수), **eX cafe만 날조**였다. 직전 두 일지가 이를 놓쳤다:
- `2026-05-31-04`는 이 15장을 "origin/main과 바이트 동일, 정상"으로 판단(날조 자체를 검증 안 함).
- `2026-05-31-07`은 "사라짐"의 렌더 메커니즘만 보고 카드를 **텍스트로 유지**(가짜 카드를 보존하는 잘못된 처방).

근본 원인: PR #88/#89에서 카페 카드를 채우며 **실제 후기를 확보하지 못하자 검색 페이지·블로그 홈 URL + 합성 설명으로 빈칸을 메웠다**(`CLAUDE.md` "출처 없는 입력 금지·URL 날조 금지" 정면 위반). 이미지 일부(당고·말차·% Arabica 컵)는 실사진이라 카드가 "그럴듯"해 보여 3개 세션이 날조를 통과시켰다.

## Decision (무엇)

**eX cafe의 blog_reviews 15장 전부 제거**한다. 카페 정보는 이미 일본어 1차 출처로 작성된 상세 페이지(`viz/excafe-review.html` ← `docs/excafe-review-translation.md`, Tabelog)가 `link`로 연결돼 있어 정보 손실이 없다.

- 사찰·관광지(죽림길·텐류지·사이호지·금각사) blog_reviews는 **실제 포스트라 보존**(정책 준수).
- 고아가 된 eX cafe 이미지 매핑 12개는 `fetch_assets.py` 재실행으로 정리(local-image-map 48→36).
- `2026-05-31-07`의 구조적 장치는 **유지**: `BlogImageSelfHostGateTests`(전수 자가호스팅 강제)·`onerror` 강화(카드 대신 이미지만 숨김). 다만 그 일지의 "텍스트 카드로 유지" 처방은 본 결정으로 대체.

채택하지 않은 대안:
- **실제 포스트 1장만 남기기**: 카페는 정책상 한국어 blog_reviews 금지이고, 그 1장의 댓글도 합성 설명이라 부적격. 전량 제거가 정답. 미채택.
- **15장을 진짜 후기로 재소싱**: 카페는 일본어 1차 출처 레인이라 한국 블로그 카드를 새로 채우는 것 자체가 정책 위반. 미채택.
- **텍스트 카드로 유지(`2026-05-31-07`)**: 가짜 후기를 보존 — 본 레포 anti-날조 규칙 위반. 미채택(본 일지가 정정).

## Consequences (그래서)

**긍정**:
- 날조 카드 제거로 **검증 가능한 콘텐츠만** 남음. 카페 정보는 Tabelog 출처 상세 페이지로 일원화.
- 사찰 후기(실제 포스트)는 보존 — 출처 정책의 종류별 레인(카페=일본어, 사찰=한국 블로그)이 데이터에 정확히 반영됨.
- "이건 다르잖아"의 정체(가짜 vs 진짜 후기) 규명·기록 — 4·7 일지의 연쇄 오진을 종결.

**부정·트레이드오프**:
- 카페 카드 15장 소멸 — 가로 스트립이 사라져 일정 카드가 단출해짐. 단 정확성 > 분량(레포 규칙).

**후속 행동**:
- **신규 blog_reviews는 사찰·관광지에만**, 그리고 **반드시 특정 포스트 URL(작성자ID/글번호) + 실제 이미지**로. 검색결과·블로그 홈 URL 금지. **게이트 추가됨**: `test_naver_blog_reviews_link_to_specific_posts`가 `search.naver`(검색 페이지)·글번호 없는 블로그 홈을 머지 차단(이번 PR). 날조 시그니처를 구조적으로 봉쇄.
- 카페·식당 정보는 `food_quality`(일본어 출처) + 번역 상세 페이지로만.

**영향 받은 파일**:
- `data/itinerary.json` (eX cafe `blog_reviews` 제거)
- `data/local-image-map.json` (고아 매핑 12개 정리)
- 본 일지
