### 2026-05-09 — 대시보드 필요성·대안 검토
- 산출물: `docs/dashboard-alternatives.md`
- 핵심 관찰:
  - 현재 `viz/dashboard.html`은 슬라이더(고가치) + 점수 입력(저가치) + localStorage(비협업)가 섞여 있음
  - 인라인 `DECISION` 데이터가 `data/decision.json`을 복제 → 수동 동기화 부담이 CLAUDE.md 규칙으로 명시될 만큼 실재
  - localStorage는 부부 각자의 브라우저에 격리되어 협업 도구로 작동하지 않음 (git/PR이 사실상 협업 채널)
  - 점수 입력 경로가 둘(JSON 편집 vs 대시보드)로 갈려 진실 출처가 모호
- 권장: **B+C 하이브리드** — 대시보드를 읽기 전용 민감도 도구로 축소하고 `score.py --sensitivity`를 추가, 인라인 데이터는 빌드 스크립트로 주입
- 보류: 슬라이더가 실제 부부 토론에 쓰일지 검증되지 않음 → 우선 `--sensitivity`만 추가하고 1회 사용 후 B 진행 여부 결정
- 다음 단계: 별도 PR로 (1) `scripts/score.py --sensitivity` 도입 → 효용 확인 후 (2) 대시보드 축소·빌드 스크립트화
