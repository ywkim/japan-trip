# 의사결정 일지

여행 결정 과정에서의 합의·보류·변경 사항을 시간순으로 기록.

## 형식

```
### YYYY-MM-DD — 주제
- 합의: ...
- 보류: ...
- 변경: ...
- 다음 단계: ...
```

## 일지

### 2026-05-09 — 레포 초기화
- 합의: 일본 여행을 정량적으로 비교하여 결정
- 합의: MCDA(가중평가) 프레임 사용 — 7개 기준, 후보 6개로 시작
- 보류: 동반자(부부만 vs +시부모), 기간, 시기, 예산
- 다음 단계: 각자 후보별 점수 입력 → 가중치 함께 조정

### 2026-05-09 — 날씨 데이터 수집·분석
- 합의: 6개 후보지 × 4개 시기(5/6/9/10월)에 대해 JMA 평년값 기반 기후 데이터 수집
- 산출물:
  - `data/weather.json` — 기온·강수·습도·강수일수·쾌적도 점수·리스크
  - `docs/weather.md` — 시기별 순위, `seasonality`/`physical_burden`에 미치는 영향
- 주요 관찰:
  - **본토 5월·10월**이 전반적으로 가장 쾌적 (오사카·교토·후쿠오카 9점)
  - **6월은 본토 장마**로 페널티 큼 → 이 시기엔 **삿포로(10)가 거의 유일한 안전지대**
  - **9월 오키나와는 태풍 최성기**(쾌적도 3) — 결항·일정 붕괴 리스크
  - **삿포로 10월은 한기 진입**(야간 한 자릿수) — 단풍 매력은 있으나 신체 부담 ↑
- 보류: 시기 확정 — 위 데이터를 기준으로 추후 합의
- 다음 단계: `decision.json`의 `seasonality` 점수를 위 표 기준으로 후보별 입력

### 2026-05-09 — 모바일에서 보는 최종 결정 요약 페이지

- 동기화 대상: `reports/final-report.md`의 결정 요약·근거·일정 → `index.html`에 정적 반영. 보고서 갱신 시 동시 갱신 (CLAUDE.md 규칙에 추가)
